# Chromium GN Print 함수 가이드

`print` 함수는 **GN 생성 시간(Gen-time)**에 콘솔에 메시지나 변수 값을 출력하는 데 사용됩니다. 빌드 로직을 디버깅하거나, 특정 시점의 변수 상태를 확인하는 데 가장 기본적인 도구입니다.

## 1. 기본 문법 (Basic Syntax)

```gn
print(value1, value2, ...)
```

*   **`value`**: 출력할 값입니다. 문자열, 정수, 불리언, 리스트, 스코프 등 모든 GN 자료형을 사용할 수 있습니다.
*   여러 인자를 전달하면 공백으로 구분되어 한 줄에 출력됩니다.

## 2. 사용 예시 (Usage Examples)

### 2.1. 변수 값 확인

가장 일반적인 사용법으로, 변수의 현재 값을 확인합니다.

```gn
my_sources = [ "a.cc", "b.cc" ]
print(my_sources)
# 출력: [ "a.cc", "b.cc" ]

is_debug = true
print("Debug mode is:", is_debug)
# 출력: Debug mode is: true
```

### 2.2. 조건문 및 반복문 디버깅

복잡한 `if` 문이나 `foreach` 반복문 내부에서 실행 흐름을 추적하고, 각 단계의 변수 값을 확인할 때 유용합니다.

```gn
foreach(i, my_list) {
  print("Current item:", i)
  if (i == "special_case") {
    print("Found special case!")
  }
}
```

### 2.3. 스코프(Scope) 내용 확인

스코프 객체(딕셔너리)를 출력하면 내부의 키-값 쌍을 모두 볼 수 있습니다.

```gn
my_scope = {
  name = "foo"
  version = 1
}
print(my_scope)
# 출력: { name = "foo" version = 1 }
```

## 3. 주의사항

*   **실행 시점**: `print`는 `gn gen` 또는 `gn args`와 같이 GN이 빌드 파일을 처리하는 시점에 실행됩니다. `ninja`를 통해 실제 컴파일이 일어나는 빌드 시점(Build-time)에는 아무것도 출력하지 않습니다.
*   **출력 형식**: 리스트는 `[ "a", "b" ]`와 같이, 스코프는 `{ a = "b" }`와 같이 GN 문법에 가까운 형식으로 출력됩니다.
*   **성능**: 너무 많은 `print` 문은 GN 생성 시간을 약간 느리게 할 수 있으므로, 디버깅이 끝나면 제거하는 것이 좋습니다.
