# Chromium GN split_list 함수 가이드

`split_list` 함수는 주어진 리스트를 지정된 개수의 하위 리스트로 분할하는 데 사용됩니다. 주로 병렬 처리가 가능한 작업들을 여러 개의 작은 그룹으로 나누어 `action` 타겟에 분배할 때 유용합니다.

## 1. 기본 문법 (Basic Syntax)

```gn
result = split_list(input, n)
```

*   **`input`**: 분할할 원본 리스트입니다.
*   **`n`**: 분할할 하위 리스트의 개수입니다.
*   **반환값**: `n`개의 하위 리스트를 포함하는 새로운 리스트를 반환합니다.

## 2. 작동 방식 (How it works)

`split_list`는 입력 리스트의 요소들을 최대한 균등하게 `n`개의 그룹으로 나눕니다.

```gn
my_list = [ 1, 2, 3, 4, 5, 6, 7, 8 ]

# 3개의 하위 리스트로 분할
result = split_list(my_list, 3)

# result는 다음과 같음:
# [
#   [ 1, 2, 3 ],
#   [ 4, 5, 6 ],
#   [ 7, 8 ],
# ]
```

## 3. 사용 예시 (Usage Example)

### 테스트 샤딩(Sharding)

많은 수의 테스트 파일을 여러 개의 테스트 실행 그룹(Shard)으로 나누어 병렬로 실행할 때 유용합니다.

```gn
# 모든 테스트 파일 목록
all_tests = [ "test1.cc", "test2.cc", "test3.cc", "test4.cc", "test5.cc" ]

# 2개의 샤드로 분할
test_shards = split_list(all_tests, 2)

# test_shards = [ [ "test1.cc", "test2.cc", "test3.cc" ], [ "test4.cc", "test5.cc" ] ]

i = 0
foreach(shard, test_shards) {
  executable("test_shard_$i") {
    testonly = true
    sources = shard
    deps = [ "//testing/gtest:gtest_main" ]
  }
  i += 1
}
```

## 4. 주의사항

*   **`n`의 값**: `n`은 1 이상이어야 합니다. `n`이 `input` 리스트의 길이보다 크면, 각 요소가 하나의 리스트에 담기고 나머지는 빈 리스트로 채워집니다.
*   **빈 리스트**: 입력 리스트가 비어있으면, `n`개의 빈 리스트를 포함하는 리스트가 반환됩니다.
