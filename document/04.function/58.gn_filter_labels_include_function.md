# Chromium GN filter_labels_include 함수 가이드

`filter_labels_include` 함수는 주어진 라벨(Label) 리스트에서 특정 패턴과 **일치하는** 라벨들만 남기고 나머지는 제거한 새로운 리스트를 반환합니다. 이는 `deps`와 같은 의존성 리스트에서 특정 유형의 타겟만 선택(Whitelist)할 때 유용합니다.

## 1. 기본 문법 (Basic Syntax)

```gn
new_list = filter_labels_include(input_list, patterns_to_include)
```

*   **`input_list`**: 필터링할 라벨들의 원본 리스트입니다.
*   **`patterns_to_include`**: 포함할 라벨 패턴 목록입니다. 와일드카드(`*`)를 사용할 수 있습니다.

## 2. 사용 예시 (Usage Examples)

### 2.1. 테스트 타겟만 선택하기

의존성 목록에서 테스트 관련 타겟들만 골라내는 경우입니다.

```gn
all_deps = [
  "//base:base",
  "//base:base_unittests",
  "//net:net",
  "//net:net_unittests",
]

# 이름에 "_unittests"가 포함된 모든 타겟만 선택
test_deps = filter_labels_include(all_deps, [ "*_unittests" ])

# test_deps는 이제 [ "//base:base_unittests", "//net:net_unittests" ]
```

### 2.2. 특정 디렉토리의 타겟만 선택하기

특정 디렉토리 하위의 모든 타겟만 의존성에 포함하고 싶을 때 사용합니다.

```gn
all_deps = [
  "//foo:lib",
  "//bar:lib",
  "//bar/baz:lib",
]

# "//bar/"로 시작하는 모든 타겟만 선택
bar_deps = filter_labels_include(all_deps, [ "//bar/*" ])

# bar_deps는 이제 [ "//bar:lib", "//bar/baz:lib" ]
```

## 3. `filter_include`와의 차이점

*   **`filter_include`**: 일반적인 문자열 리스트(주로 파일 경로)를 필터링합니다.
*   **`filter_labels_include`**: 타겟 라벨(`//path:name`) 리스트를 필터링합니다. 패턴은 전체 라벨 문자열에 대해 매칭됩니다.

## 4. `filter_labels_exclude`와의 관계

`filter_labels_include`는 패턴과 일치하는 라벨을 제거하는 `filter_labels_exclude`와 정반대의 기능을 수행합니다.
