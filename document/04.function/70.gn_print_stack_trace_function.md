# Chromium GN print_stack_trace 함수 가이드

`print_stack_trace` 함수는 GN 생성 시간(Gen-time)에 현재 위치의 호출 스택(Call Stack)을 콘솔에 출력하는 데 사용됩니다. 복잡한 템플릿(Template)이나 `import` 구조에서 현재 코드가 어떤 경로를 통해 호출되었는지 추적하는 데 매우 유용한 디버깅 도구입니다.

## 1. 기본 문법 (Basic Syntax)

```gn
print_stack_trace()
```

이 함수는 별도의 인자를 받지 않습니다.

## 2. 사용 예시 (Usage Examples)

### 템플릿 호출 경로 추적

특정 템플릿이 예상치 못한 곳에서 호출되거나, 의도하지 않은 인자로 호출될 때 그 경로를 파악하기 위해 사용합니다.

```gn
# //build/templates/my_template.gni
template("my_template") {
  # 템플릿이 호출될 때마다 호출 스택을 출력
  print_stack_trace()

  # ...
}
```

**출력 예시:**

`gn gen` 실행 시 다음과 같은 스택 트레이스가 출력될 수 있습니다.

```text
Stack trace:
  //build/templates/my_template.gni:3:3: In my_template
  //chrome/browser/BUILD.gn:105:1: From here
  //chrome/BUILD.gn:20:5: From here
```

위 출력은 `//chrome/BUILD.gn`의 20번째 줄에서 `//chrome/browser/BUILD.gn`의 타겟을 의존하고, 그 파일의 105번째 줄에서 `my_template`이 호출되었음을 보여줍니다.

## 3. `print` 및 `assert`와의 조합

`print_stack_trace`는 다른 디버깅 함수와 함께 사용될 때 더욱 강력합니다.

```gn
if (some_unexpected_condition) {
  print("Unexpected condition met with value:", some_variable)
  print_stack_trace()
  assert(false, "Stopping build to investigate.")
}
```

## 4. 주의사항

*   **실행 시점**: `print` 함수와 마찬가지로, `print_stack_trace`는 `gn gen` 시점에 실행됩니다. 실제 컴파일이 일어나는 빌드 시점(Build-time)에는 아무런 동작도 하지 않습니다.
*   **디버깅용**: 이 함수는 디버깅 목적으로만 사용해야 하며, 최종 코드에는 남겨두지 않는 것이 좋습니다.
