# Chromium GN filter_include 함수 가이드

`filter_include` 함수는 주어진 리스트에서 특정 패턴과 **일치하는** 요소들만 남기고 나머지는 제거한 새로운 리스트를 반환합니다. `filter_exclude`의 반대 기능을 수행하며, 화이트리스트(Whitelist) 방식으로 리스트를 필터링할 때 사용됩니다.

## 1. 기본 문법 (Basic Syntax)

```gn
new_list = filter_include(input_list, patterns_to_include)
```

*   **`input_list`**: 필터링할 원본 리스트입니다.
*   **`patterns_to_include`**: 포함할 패턴 목록입니다. 와일드카드(`*`)를 사용할 수 있습니다.

## 2. 사용 예시 (Usage Examples)

### 2.1. 특정 확장자 파일만 골라내기

소스 파일 목록에서 C++ 소스 파일(`.cc`)만 추려내고 싶을 때 사용합니다.

```gn
all_files = [
  "foo.cc",
  "foo.h",
  "bar.cc",
  "bar.h",
  "README.txt",
]

# .cc 파일만 포함
sources = filter_include(all_files, [ "*.cc" ])

# sources는 이제 [ "foo.cc", "bar.cc" ]
```

### 2.2. 특정 플랫폼 파일만 선택하기

여러 플랫폼용 파일이 섞여 있는 리스트에서 현재 빌드 대상 플랫폼에 맞는 파일만 선택할 때 유용합니다.

```gn
mixed_sources = [
  "device_linux.cc",
  "device_mac.mm",
  "device_win.cc",
  "device_shared.cc",
]

if (is_linux) {
  # Linux 관련 파일과 공용 파일만 선택
  linux_sources = filter_include(mixed_sources, [ "*_linux.cc", "*_shared.cc" ])
  # linux_sources는 [ "device_linux.cc", "device_shared.cc" ]
}
```

## 3. `filter_exclude`와의 관계

`filter_include`는 `filter_exclude`와 상호 보완적입니다.

*   **`filter_include`**: 패턴에 **일치하는** 것만 남김 (화이트리스트 방식).
*   **`filter_exclude`**: 패턴에 **일치하는** 것을 제거 (블랙리스트 방식).

## 4. 활용 팁

`get_target_outputs` 등으로 얻은 파일 목록에서 특정 타입의 파일(예: 객체 파일 `.o` 또는 특정 헤더 `.h`)만 후속 작업(`action` 등)의 입력으로 사용하고 싶을 때 자주 사용됩니다.
