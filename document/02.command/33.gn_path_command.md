# Chromium GN Path 명령어 가이드

`gn path`는 두 타겟(Target) 사이의 의존성 경로를 찾아주는 명령어입니다. "타겟 A가 왜 타겟 B를 의존하고 있지?"라는 의문이 들 때, 그 연결 고리를 파악하는 데 매우 유용합니다.

## 1. 기본 사용법 (Basic Usage)

```bash
gn path <out_dir> <target_from> <target_to>
```

*   **`<out_dir>`**: 빌드 디렉토리 경로 (예: `out/Default`).
*   **`<target_from>`**: 의존성 경로의 시작점 (상위 타겟).
*   **`<target_to>`**: 의존성 경로의 도착점 (하위 타겟).

### 예시

```bash
# //chrome 타겟이 //third_party/icu를 왜 포함하는지 확인
gn path out/Default //chrome //third_party/icu
```

## 2. 출력 해석 (Output Interpretation)

명령어를 실행하면 타겟들의 리스트가 출력됩니다.

```text
//chrome:chrome --[private]-->
//chrome/browser:browser --[public]-->
//net:net --[public]-->
//third_party/icu:icuuc
```

*   위 출력은 `//chrome`이 `//chrome/browser`를 의존하고, 그것이 `//net`을 의존하며, 최종적으로 `//third_party/icu`를 의존한다는 것을 의미합니다.
*   화살표 위의 `private`, `public`, `data`는 의존성 유형(`deps`, `public_deps`, `data_deps`)을 나타냅니다.

## 3. 주요 옵션 (Options)

### `--all`
기본적으로 `gn path`는 가장 짧은 경로 하나만 보여줍니다. 두 타겟 사이에 존재하는 **모든** 경로를 보고 싶다면 이 옵션을 사용합니다.

```bash
gn path out/Default //chrome //base --all
```

### `--public`
`public_deps`로만 연결된 경로를 찾습니다. 이는 헤더 파일 포함(include) 관계를 추적할 때 유용합니다. 만약 A가 B를 public하게 의존하지 않는다면, A를 사용하는 쪽에서는 B의 헤더를 사용할 수 없기 때문입니다.

```bash
gn path out/Default //my_lib //base --public
```

## 4. 활용 사례 (Use Cases)

*   **불필요한 의존성 제거**: 바이너리 크기를 줄이기 위해 특정 라이브러리가 왜 링크되는지 추적할 때.
*   **순환 참조 디버깅**: 의존성 사이클이 발생했을 때 경로를 파악하기 위해.
*   **가시성(Visibility) 확인**: `gn check` 오류가 발생했을 때, 의존성 경로가 올바르게 설정되어 있는지 확인할 때.
