# Chromium GN Args 명령어 가이드

`gn args`는 빌드 디렉토리의 빌드 인자(Build Arguments)를 설정하고 확인하는 명령어입니다. GN 빌드 시스템의 유연성은 대부분 이 명령어를 통해 제어됩니다.

## 1. 기본 사용법 (Basic Usage)

가장 기본적인 사용법은 빌드 디렉토리를 지정하여 실행하는 것입니다.

```bash
gn args out/Default
```

이 명령어를 실행하면 다음과 같은 과정이 진행됩니다.
1.  시스템의 기본 텍스트 편집기(환경변수 `EDITOR` 또는 `VISUAL`에 지정된 편집기)가 열립니다.
2.  사용자는 `args.gn` 파일 내용을 수정할 수 있습니다.
3.  편집기를 저장하고 닫으면, GN은 변경된 설정을 바탕으로 빌드 파일(`build.ninja`)을 자동으로 재생성합니다.

## 2. `args.gn` 파일 편집

편집기가 열리면 GN 문법을 사용하여 변수 값을 설정할 수 있습니다.

```gn
# args.gn 예시
is_debug = false
is_component_build = true
target_cpu = "x64"
symbol_level = 0
```

*   **`is_debug`**: 디버그 모드 여부 (`true`/`false`).
*   **`is_component_build`**: 공유 라이브러리(DLL/SO) 빌드 여부. 개발 속도 향상을 위해 주로 `true`로 설정합니다.
*   **`target_cpu`**: 타겟 아키텍처 (`"x64"`, `"arm64"` 등).

## 3. 인자 목록 확인 (`--list`)

현재 설정에서 사용 가능한 모든 빌드 인자와 그 기본값, 현재 설정값, 그리고 설명(Documentation)을 보고 싶을 때 사용합니다.

```bash
gn args out/Default --list
```

특정 인자만 검색하고 싶다면 `--list` 뒤에 인자 이름을 지정할 수 있습니다.

```bash
gn args out/Default --list=is_debug
gn args out/Default --list --short  # 설명 제외하고 값만 출력
```

## 4. 커맨드 라인 오버라이드

CI(Continuous Integration) 환경이나 스크립트에서 편집기를 열지 않고 인자를 설정해야 할 때는 `gn gen` 명령의 `--args` 플래그를 사용합니다.

```bash
gn gen out/Release --args="is_debug=false enable_nacl=false"
```

*   여러 인자는 공백으로 구분합니다.
*   문자열 값은 이스케이프 처리된 따옴표(`\"`)를 사용해야 할 수 있습니다.

## 5. 생성된 `args.gn` 파일 위치

`gn args`로 설정한 내용은 빌드 디렉토리 내의 `args.gn` 파일에 저장됩니다.

*   경로: `out/Default/args.gn`

이 파일을 직접 수정하고 `gn gen out/Default`를 실행해도 동일한 효과를 냅니다.
